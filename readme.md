# DFNAS-LLM: Neural Architecture Search Enhanced by Large Language Models and Discrete Fireworks Algorithm

DFNAS-LLM is an efficient Neural Architecture Search (NAS) framework that integrates the Discrete Fireworks Algorithm (DFWA) with Large Language Model (LLM) based surrogate models. By leveraging the zero-shot reasoning capabilities of LLMs to predict network potential from early training signals (Early Stopping), this framework achieves a 78% reduction in computational costs while maintaining competitive search quality.


## 🌟 Key Features

- **Discrete Fireworks Algorithm (DFWA)**: An optimization algorithm specifically tailored for binary search spaces, featuring discrete explosion/mutation operators and an archive mechanism to prevent redundant evaluations.
- **LLM-powered Surrogate Evaluation**: Utilizes state-of-the-art LLMs (e.g., DeepSeek-V3, GPT-4o, Gemini) to rank candidate architectures based on initial training trajectories.
- **Efficient Encoding Scheme**: A 72-bit binary encoding representing node types (MBConv modules), DAG-based topological connections, and discrete channel growth configurations.
- **Resource Efficiency**: Discovered architectures achieve high accuracy on CIFAR-10/100 with only ~0.7M parameters.



## 🚀 Getting Started

### 1. Prerequisites

Ensure you have Python, Numpy  PyTorch installed.


### 2. Configure LLM API

Configure your API Key in `llm/api.py`. Supported models include `v3` (DeepSeek-V3), `r1` (DeepSeek-R1), `gpt-4o`, etc.

```python
# In llm/api.py
self.token = 'YOUR_API_KEY'
self.url = 'YOUR_API_ENDPOINT'
```

### 3. Run Search

Start the neural architecture search using the following command:

```bash
# Search on CIFAR-10 using LLM proxy
python run.py --alg DFWA --bench cifar10 --proxy --gpu 0 --name "experiment_01"

# Arguments:
# --alg: Algorithm name (e.g., DFWA)
# --bench: Benchmark dataset (cifar10, cifar100)
# --proxy: Enable LLM surrogate model
# --dim: Encoding dimension (default: 72)
# --max_eval: Maximum number of evaluations
# --stop_epoch: Full training epochs
# --pre_epoch: Early stopping epochs for proxy (default: 6)
```

## 🧠 Methodology

DFNAS-LLM follows a two-stage evaluation strategy:
1. **Early Training**: Candidates generated by DFWA are trained for a few epochs ($T_{pre}$).
2. **LLM Ranking**: Training/validation accuracy curves are formatted into a prompt and sent to an LLM.
3. **Selective Training**: Only the Top-$k$ candidates predicted by the LLM undergo full training ($T_{full}$) to update the population.

## 📊 Experimental Results

Experiments on CIFAR-10/100 demonstrate that DFNAS-LLM discovers highly efficient architectures with minimal parameters (~0.7M) while significantly outperforming traditional NAS methods in terms of search speed and resource consumption.
